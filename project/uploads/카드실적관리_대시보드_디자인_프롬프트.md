## 프롬프트 본문

아래 PRD를 기반으로 "카드실적관리 대시보드"를 디자인해줘.

### 1. 서비스 개요
- 사용자는 여러 개의 신용/체크카드를 보유하고, 카드별 혜택(캐시백, 포인트, 할인 등)을 받기 위한 전월 실적 조건을 관리해야 함
- 카드사 이용내역(엑셀)을 업로드하면 실적 포함 여부·혜택 대상 여부를 자동 판정하고, 카드별 실적 진행 상황과 혜택 획득 현황을 관리하는 도구
- 성공 기준: 사용자가 메인 대시보드만 봐도 "이번 달 각 카드의 실적 기준 충족 여부"를 즉시, 정확히 확인할 수 있어야 함

### 2. 사용 환경 (중요 — 필독)
- **담당자(사용자)는 컴퓨터 모니터를 통해 주로 사용한다.** 모바일 대응은 우선순위가 아니며, **데스크탑(와이드 스크린) 환경을 기준으로 설계**할 것
- 넓은 화면을 활용해 좌측 사이드 내비게이션 + 우측 콘텐츠 영역 같은 멀티 컬럼 레이아웃을 적극 사용
- 카드 여러 장을 카드형 그리드(예: 3~4열)로 동시에 비교할 수 있도록 구성
- 표/리스트형 데이터(거래 내역 등)는 한 화면에서 스크롤 없이 많은 행을 볼 수 있도록 여백과 폰트 크기를 데스크탑 뷰잉 거리에 맞게 설정 (모바일처럼 과도하게 크게 만들지 않음)
- 마우스 hover, 클릭 등 데스크탑 인터랙션(호버 시 상세 툴팁, 테이블 행 hover 강조 등)을 적극 활용

### 3. 디자인 원칙 (첨부 이미지 참조)
첨부한 사내 포털 팝업 이미지(한화솔루션 우리사주 청약 안내 팝업)의 톤앤매너를 참고해서 아래 원칙을 따라줘.

- **컬러**: 기본 배경은 화이트, 포인트 컬러는 오렌지 계열 단색 하나만 사용. 오렌지는 핵심 강조 요소(헤드라인 텍스트, CTA 버튼, 상태 배지/진행률 바 등)에만 제한적으로 사용하고 그 외에는 블랙/그레이 톤의 절제된 팔레트를 유지
- **타이포그래피 위계**: 가장 중요한 정보(실적 달성 여부, 핵심 금액 등)는 굵고 큰 오렌지색 텍스트로 강조, 본문/부가 정보는 작은 검정/회색 텍스트로 차분하게 처리 → 한 화면 안에서 "제일 먼저 봐야 할 것"이 시각적으로 명확히 구분되어야 함
- **레이아웃**: 얇은 보더(border)와 넉넉한 여백을 가진 화이트 카드/패널 구조. 그림자는 과하지 않게, 미니멀하고 정돈된 느낌
- **버튼/CTA**: 모서리가 둥근 사각형, 오렌지 단색 채움, 흰색 굵은 텍스트, 중앙 정렬 — 예: "실적 확인하기", "엑셀 업로드" 등 주요 액션 버튼에 적용
- **부가 정보 텍스트**: 체크박스, 타임스탬프, 안내 문구 등은 작은 회색 텍스트로 하단이나 여백에 절제되게 배치
- **전체 톤**: 장식적이거나 화려한 느낌이 아닌, 신뢰감 있고 정돈된 **기업용/사내 시스템(Corporate/Enterprise) 톤**을 유지할 것. 채도가 높은 다색 배색이나 그라데이션은 지양

### 4. 화면 구성 (4개 화면)

**① 메인 대시보드**
- 카드별로 카드명/카드사, 이번 달 사용금액, 실적 인정 금액, 실적 기준 금액 대비 진행률(프로그레스 바), 달성/미달성/근접 상태 배지, 획득 혜택금액 합계, 받은 혜택 목록 요약을 카드형 UI로 표시
- 상단에 조회 월 선택, 전체 카드 합계 요약(총 사용금액, 실적 달성 카드 수/전체, 총 혜택금액) 노출
- 실적 미달 카드는 시각적으로 상단 강조(경고성 컬러 포인트, 오렌지 계열 내에서 처리)
- 카드 클릭 시 ③번 화면으로 드릴다운

**② 카드내역 입력 (엑셀 업로드)**
- 카드 선택 → 엑셀 드래그앤드롭/업로드 영역
- 업로드 후 컬럼 매핑 확인 UI, 업로드 결과 요약(총 건수/자동 매칭 성공/실패 건수)을 테이블 또는 요약 카드로 표시

**③ 카드별 사용내역 관리**
- 카드 선택 후 거래 내역을 테이블 형태로 표시 (거래일자, 가맹점명, 금액, 결제방식)
- 각 행마다 실적 포함 여부(토글), 혜택 대상 여부 및 혜택 항목(드롭다운)을 인라인으로 수정 가능
- 자동판정/수동수정 여부를 작은 배지로 구분 표시
- 상단 필터(월별, 실적포함여부, 혜택여부, 미분류만 보기), 검색창 제공

**④ 카드 설정**
- 카드 등록/수정 폼 (카드명, 카드사, 색상 등)
- 전월 실적 기준 금액 입력
- **실적 산정 기간 설정** (예: 매월 1일~말일 / 전월 21일~당월 20일 등 시작일·종료일 지정 UI)
- 혜택 항목 리스트 등록/수정 (혜택처, 혜택 내용, 매칭 조건, 실적 포함 여부 토글) — 카드 하나에 여러 혜택 항목을 추가/삭제할 수 있는 반복 입력 UI
- 월별 실적 이력 조회 섹션(과거 월별 실적 인정 금액/달성 여부 리스트 또는 미니 차트)

### 5. 산출물 요청
- 위 4개 화면에 대해 데스크탑 해상도(예: 1440px 폭 기준) 기준의 고해상도 UI 목업(또는 인터랙티브 프로토타입)을 제작
- 화면 간 이동(사이드 내비게이션 또는 상단 탭)이 느껴지도록 공통 레이아웃 프레임을 먼저 정의한 뒤 각 화면 적용
- 색상은 오렌지 1색 + 그레이스케일 조합으로 제한하고, 상태 표현(달성/미달성/경고)은 오렌지의 명도·채도 변형이나 아이콘으로 구분
